# Loxion Movement Engineering Principles

## 1. Build → Test → Commit → Document

Every feature follows this cycle.

## 2. Owner First

The admin experience is as important as the customer experience.

## 3. Free Tier Friendly

Prefer free services and efficient queries.

## 4. One Responsibility Per Component

UI renders.
Services perform work.
Pages compose features.

## 5. Reuse Before Rewrite

If the same code appears twice, consider extracting it.

## 6. Simplicity Wins

Choose the simplest solution that solves today's problem well.
